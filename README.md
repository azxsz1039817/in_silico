# In Silico Sociology Replication Code

This directory contains the cleaned replication package for the In Silico Sociology project. It reruns the empirical analyses from cached GPT-3 response/projection data. The original GPT-3 data-collection step is preserved only as archival code because the original `davinci` API/model context cannot be faithfully recreated today.

## Directory Layout

- `data/raw/prompts/`: canonical prompt metadata and semantic-axis label metadata.
- `data/raw/responses/`: compressed cached GPT-3 completion/projection datasets. These include full generated opinion completions and full prompts.
- `data/raw/justifications/`: compressed full-text justification data plus split float32 justification embedding archives.
- `data/processed/`: analysis-ready datasets generated by the replication scripts.
- `scripts/`: Python scripts for the replication workflow.
- `tables/`: coefficient tables, binomial tests, consistency checks, and justification proportion tests.
- `figures/`: regenerated Python figures.
- `docs/`: analysis map, data dictionary, and reproducibility notes.
- `tests/`: smoke tests for script importability, command-line help, required inputs, and secret/path scans.

## Step-By-Step Replication

Run all commands from `clean_codebase/`.

### 1. Install Dependencies

```bash
pip install -r requirements.txt
```

The archival OpenAI scripts require an `OPENAI_API_KEY`, but they are not part of the default replication workflow.

### 2. Unpack Compressed Raw Data

```bash
python scripts/unpack_raw_data.py
```

This creates local working CSVs from:

- `data/raw/responses/covid_response_projections.csv.gz`
- `data/raw/responses/validation_response_projections.csv.gz`
- `data/raw/justifications/justification_texts.csv.gz`

The `.gz` files are the repository-tracked raw data files. The unpacked `.csv` files are local working copies. `scripts/run_all.py --skip-api` runs this unpack step automatically.

### 3. Normalize Prompt Metadata

```bash
python scripts/build_prompt_tables.py
```

This reads the prompt metadata in `data/raw/prompts/` and writes normalized prompt/label tables to `data/processed/`.

### 4. Assemble Cached GPT-3 Response Data

```bash
python scripts/assemble_responses.py --dataset covid
python scripts/assemble_responses.py --dataset validation
```

These scripts read the unpacked response/projection CSVs and write:

- `data/processed/covid_responses.csv`
- `data/processed/covid_responses_named.csv`
- `data/processed/validation_responses.csv`
- `data/processed/validation_responses_named.csv`

The `*_named.csv` files restrict to the named partisan primes used in the manuscript coefficient plots.

### 5. Reproduce Coefficient Tables And Figures

```bash
python scripts/coef_plots.py --dataset covid
python scripts/coef_plots.py --dataset validation
```

These scripts estimate the liberal-prime coefficient for each prompt wording and write coefficient tables and faceted manuscript-style figures. Validation outputs are split into:

- `figures/validation_coefficients_part1.png`
- `figures/validation_coefficients_part2.png`

Mixed-axis figures label each facet with its own semantic-axis contrast, such as `good idea - bad idea`, `safe - dangerous`, or `lab - animal`.

### 6. Reproduce Forecast And Justification Tests

```bash
python scripts/stat_tests.py
```

This writes:

- `tables/forecast_binomial_tests.csv`
- `tables/justification_two_proportion_tests.csv`
- `tables/stat_tests_summary.json`

The justification proportion tests reproduce the original R `prop.test(..., alternative = "greater", correct = FALSE)` comparisons from the justification section and appendix.

### 7. Validate Forecast Correct/Wrong/No-Effect Counts

```bash
python scripts/validate_consistency.py
```

This writes:

- `tables/consistency_checks.csv`
- `tables/covid_coefficients_with_consistency_check.csv`
- `tables/consistency_checks_notes.json`

The consistency check reproduces the manuscript count of 109 correct, 12 wrong, and 58 no-effect prompt-level forecasts.

### 8. Summarize Justification Cache Metadata And Bar Charts

```bash
python scripts/cluster_justifications.py
python scripts/justification_figures.py
```

The package includes full justification text in `justification_texts.csv.gz`, split float32 embedding archives in `justification_embeddings_float32_part*.npz`, and `justification_embeddings_manifest.csv` for slice lookup. `cluster_justifications.py` summarizes the cache by default. To cluster one prompt/sentiment pair from the included embeddings, pass `--prompt-no`, `--connector`, and `--sentiment`.

`justification_figures.py` writes manuscript-style stacked liberal/conservative bar charts:

- `figures/justification_vaccine_bars.png`
- `figures/justification_mask_mandate_bars.png`
- `figures/justification_fear_virus_bars.png`
- `figures/justification_appendix_bars.png`

### 9. Run The Whole Cached-Data Pipeline

```bash
python scripts/run_all.py --skip-api
```

This executes steps 2 through 8 in order. The `--skip-api` flag is included for clarity; API-generating scripts are never called by default.

### 10. Optional Smoke Tests

```bash
python -m pytest -q
```

Without `pytest`, basic checks can still be run manually:

```bash
python -m compileall -q scripts tests
python scripts/run_all.py --skip-api
```

## Archival API Scripts

The scripts named `archive_*` preserve the historical OpenAI API style for provenance. They are not required for reproducing the analyses and should not be expected to recreate the original data exactly.

## Non-Obvious Coding Notes

- The compressed response CSVs are canonical analysis inputs. They preserve full generated completions and full prompts in normalized tabular form.
- The original per-prompt output files were removed because they were intermediate cache files and created most of the file-count clutter.
- The original justification embedding pickle cache was converted to split float32 NPZ files. Each chunk is under GitHub's 100 MB per-file limit, so Git LFS is not required.
- The correct/wrong/no-effect coding cannot be inferred from `reverse_code` alone for every prompt. `validate_consistency.py` includes keyword-level overrides for business and school open/closed variants.
- `keeping schools open` remains expected conservative. Three liberal-observed `keeping schools open` rows are counted as wrong, matching the manuscript count.
